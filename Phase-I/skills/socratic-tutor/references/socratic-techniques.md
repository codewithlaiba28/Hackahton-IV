# Socratic Questioning Techniques

## The Six Types of Socratic Questions

### 1. Clarification Questions

**Purpose:** Help students clarify their thinking

```
Examples:
- "What do you mean by...?"
- "Can you put that another way?"
- "What's the main point you're making?"
- "How does this relate to our problem?"
- "Can you give me an example?"

When to use:
- Student's statement is vague
- Need to ensure mutual understanding
- Student seems confused about their own thinking
```

### 2. Assumption-Probing Questions

**Purpose:** Examine underlying assumptions

```
Examples:
- "What are you assuming here?"
- "Why would someone make this assumption?"
- "What if we assumed the opposite?"
- "Is this assumption always valid?"
- "How did you arrive at this assumption?"

When to use:
- Student's reasoning has hidden assumptions
- Assumption may be incorrect
- Need to examine foundations of argument
```

### 3. Evidence-Probing Questions

**Purpose:** Connect claims to evidence

```
Examples:
- "What evidence supports this?"
- "How do you know this is true?"
- "What would change your mind?"
- "Where did you find this information?"
- "What data are you basing this on?"

When to use:
- Student makes unsupported claims
- Need to distinguish opinion from fact
- Building critical evaluation skills
```

### 4. Perspective Questions

**Purpose:** Consider alternative viewpoints

```
Examples:
- "What would someone who disagrees say?"
- "How might an expert view this?"
- "What's another way to look at this?"
- "How would this look from [different context]?"
- "What are the strengths and weaknesses of this view?"

When to use:
- Student has narrow perspective
- Need to develop flexible thinking
- Multiple valid approaches exist
```

### 5. Implication Questions

**Purpose:** Explore consequences and implications

```
Examples:
- "What are the implications of this?"
- "If this is true, what else follows?"
- "What would happen if we applied this?"
- "How does this affect...?"
- "What are the long-term consequences?"

When to use:
- Student hasn't considered consequences
- Need to connect to broader context
- Building systems thinking
```

### 6. Meta-Questions (About the Question)

**Purpose:** Develop metacognitive awareness

```
Examples:
- "Why do you think I asked that question?"
- "What was your thought process?"
- "How did you arrive at your answer?"
- "What strategy did you use?"
- "What would you do differently next time?"

When to use:
- After problem is solved
- Building metacognitive skills
- Consolidating learning
```

## Hint Hierarchy Examples

### Domain: Programming

```
Problem: "How do I reverse a string in Python?"

Level 1 (Metacognitive):
"What strategies do you know for manipulating strings?"

Level 2 (General):
"Think about operations you can perform on sequences."

Level 3 (Specific):
"Python has built-in functions for sequences. What might apply here?"

Level 4 (Content):
"Remember slicing notation [start:stop:step]?"

Level 5 (Near-solution):
"What if you used [::-1] as the slice?"
```

### Domain: Mathematics

```
Problem: "How do I solve 2x + 5 = 15?"

Level 1 (Metacognitive):
"What's your goal when solving an equation?"

Level 2 (General):
"What operations can you use to isolate x?"

Level 3 (Specific):
"What's the first step to get x by itself?"

Level 4 (Content):
"What happens if you subtract 5 from both sides?"

Level 5 (Near-solution):
"After subtracting 5, you have 2x = 10. What's next?"
```

### Domain: Science

```
Problem: "Why does ice float on water?"

Level 1 (Metacognitive):
"What factors determine whether something floats?"

Level 2 (General):
"Think about the relationship between density and buoyancy."

Level 3 (Specific):
"How does the density of ice compare to liquid water?"

Level 4 (Content):
"Remember what happens to water molecules when they freeze?"

Level 5 (Near-solution):
"Ice has a crystalline structure with more space between molecules. What does this mean for density?"
```

## Common Student Misconceptions

### Misconception: "I need to get it right immediately"

```
Response:
"Mistakes are how our brains learn! Every wrong attempt
teaches you something. What matters is thinking through it."

Strategy:
- Normalize struggle
- Celebrate productive errors
- Share stories of famous failures
```

### Misconception: "I'm just not good at this"

```
Response:
"This is a skill that can be learned, not a talent you're born with.
Everyone who's good at this started where you are now."

Strategy:
- Growth mindset messaging
- Point to student's own progress
- Break into achievable steps
```

### Misconception: "The tutor should just tell me"

```
Response:
"I understand you want the answer. My job isn't to give you fish,
but to teach you to fish. You'll learn much more by figuring it out."

Strategy:
- Explain the pedagogy
- Show value of discovery
- Give satisfying hints
```

### Misconception: "There's only one right way"

```
Response:
"Great observation! There are actually multiple approaches.
You found one. Can you think of another way?"

Strategy:
- Validate their approach
- Encourage alternative methods
- Compare different strategies
```

## Problem-Solving Heuristics

### Polya's Four Steps

```
1. Understand the Problem
   - What is given?
   - What is unknown?
   - What are the conditions?

2. Devise a Plan
   - Have you seen a similar problem?
   - Can you restate the problem?
   - Can you solve a simpler version?

3. Carry Out the Plan
   - Execute your strategy
   - Check each step
   - Adjust if needed

4. Look Back
   - Does the answer make sense?
   - Is there another way?
   - What did you learn?
```

### Debugging Heuristics (Programming)

```
1. Reproduce the error consistently
2. Read the error message carefully
3. Check the line mentioned (and surrounding lines)
4. Add print statements to trace execution
5. Isolate the problem (remove code until it works)
6. Search for similar issues
7. Explain the problem to someone (rubber duck)
8. Take a break and come back fresh
```

### Scientific Reasoning

```
1. Observe phenomena
2. Ask a question
3. Form a hypothesis
4. Make a prediction
5. Test the prediction
6. Analyze results
7. Draw conclusions
8. Communicate findings
```

## Metacognitive Prompts

### Before Problem-Solving

```
- "What type of problem is this?"
- "What do you already know that might help?"
- "What's your plan for solving this?"
- "How long do you think it will take?"
- "What resources might you need?"
```

### During Problem-Solving

```
- "Is your plan working?"
- "What's telling you that?"
- "Should you try a different approach?"
- "What part is most challenging?"
- "Are you making progress?"
```

### After Problem-Solving

```
- "How did you figure it out?"
- "What was the key insight?"
- "What would you do differently next time?"
- "How could you use this approach elsewhere?"
- "What did you learn about your own thinking?"
```

## Encouragement Phrases

### For Effort

```
- "I can see you're really thinking about this."
- "You're working hard on this - that's great!"
- "I appreciate how you're sticking with it."
- "Your persistence is paying off!"
```

### For Strategy

```
- "That's a smart approach!"
- "I like how you broke it down."
- "Great use of [specific technique]!"
- "That's exactly how an expert would think about it."
```

### For Progress

```
- "Look how far you've come!"
- "You've figured out the hard part!"
- "You're on the right track!"
- "That insight shows real understanding!"
```

### For Struggle

```
- "This is challenging - that's why it's worth learning."
- "Feeling stuck means your brain is growing."
- "The struggle is where learning happens."
- "Don't give up - you're closer than you think!"
```

## Fading Support Strategy

### Stage 1: High Support (I Do)

```
Tutor behavior:
- Model thinking aloud
- Explain each step
- Show expert approach
- Student observes

Student role:
- Watch and listen
- Ask clarifying questions
- Notice patterns
```

### Stage 2: Shared Responsibility (We Do)

```
Tutor behavior:
- Prompt student to contribute
- Fill in gaps as needed
- Ask guiding questions
- Provide hints

Student role:
- Participate actively
- Try steps with support
- Ask for help when stuck
```

### Stage 3: Low Support (You Do)

```
Tutor behavior:
- Student leads, tutor follows
- Minimal hints only
- Ask meta-questions
- Celebrate insights

Student role:
- Solve independently
- Self-monitor progress
- Request specific help
```

### Stage 4: Independence

```
Tutor behavior:
- Present similar problem
- Step back completely
- Observe and assess
- Confirm success

Student role:
- Solve completely alone
- Self-correct errors
- Apply learned strategies
```

## Quality Indicators

### Good Socratic Tutoring

```
✓ Student does most of the thinking
✓ Student articulates key insights
✓ Hints preserve cognitive work
✓ Struggle is productive, not frustrating
✓ Student can explain reasoning
✓ Learning transfers to new problems
✓ Student confidence increases
```

### Poor Socratic Tutoring

```
✗ Tutor gives answers too quickly
✗ Hints are too specific (give away solution)
✗ Student is passive
✗ Struggle becomes frustrating
✗ Focus is on answer, not process
✗ No consolidation of learning
✗ Student can't explain how they got it
```
